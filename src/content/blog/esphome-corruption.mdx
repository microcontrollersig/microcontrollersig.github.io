This guide this for installing esptool and esphome using uv python package and library manager

If you don't want to install uv (because you want keep your python install), then this guide is NOT for you.

Here are steps to follow when can't upload ESPHome to your esp32.

## Prerequisites

### Is python installed?

```sh
python --version
```

### If python is not installed

Two options, install python manually, or use **uv**

This guide only shows you how to install python using uv.

Install uv: https://docs.astral.sh/uv/getting-started/installation/

### OK, I've installed uv, now what

```sh
uv tool install esptool
uv tool install esphome --with fatfs-ng --with littlefs-python --with pip --with wheel
```

To check what is installed:

```sh
uv tool list
```

### Make python version default

```sh
uv python pin --global 3.14
```

Also check order of python in your PATH:

On windows CMD:

```sh
where.exe python
```


<Tip>
<h3>Python Libraries</h3> <p>Official python manager for downloading libraries is called **pip**, and the official registry for libraries is **pypi** . The pypi page for esptool is <HyperLink href="https://pypi.org/project/esptool">here</HyperLink>  and esphome is <HyperLink href="https://pypi.org/project/esphome">here</HyperLink></p>
</Tip>
## EXECUTIVE SUMMARY OF PROCEDURE

### Erase Flash 
   ```sh 
   esptool erase_flash --device COM7
   ```
### Upload firmware via USB 
  Need to upload esphome yaml via USB after erase. 
  
  Once esphome is on device, over-the-air upload will work.

  ```sh
   esphome compile something.yaml
   ```

   Followed by:

   ```sh
   esphome upload something.yaml --device COM7
   ```

Now uploading from Home Assistant ESPHome Addon should work.
